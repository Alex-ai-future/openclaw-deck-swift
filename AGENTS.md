# AGENTS.md - 项目工作规则

**项目：** OpenClaw Deck Swift  
**位置：** `~/Projects/openclaw-deck-swift/`

---

## 🛑 核心规则：修改前必须确认

**在任何修改操作之前，必须遵循以下流程：**

```
1. 分析需求 → 2. 制定计划 → 3. 向用户确认 → 4. 获得批准 → 5. 执行修改
```

### 具体要求：

1. **不要直接修改文件**
   - 即使是很小的改动，也要先告知用户
   - 说明你要改什么、为什么改、怎么改

2. **提供完整的修改计划**
   - 列出所有要修改的文件
   - 说明每处修改的内容
   - 解释修改的原因和预期效果

3. **等待用户批准**
   - 用户说"可以"、"好的"、"批准"等明确表示后，才能执行
   - 如果用户有疑问，先解答再继续

4. **修改后汇报**
   - 完成后告知用户修改已完成
   - 如有需要，说明如何验证修改效果

---

## 📋 工作流程

### 进入项目时：
- [ ] 读取本 `AGENTS.md` 文件
- [ ] 了解项目结构和当前状态
- [ ] 等待用户指令

### 收到任务时：
- [ ] 分析任务需求
- [ ] 制定详细计划
- [ ] **向用户确认计划** ⚠️
- [ ] 获得批准后执行
- [ ] 完成后汇报

### 日常操作：
- [ ] 读取文件、探索代码 → ✅ 可以直接做
- [ ] 修改文件、删除文件、创建文件 → ⚠️ **必须先确认**
- [ ] 运行构建/测试命令 → ⚠️ **必须先确认**
- [ ] 执行 git 操作（commit/push 等）→ ⚠️ **必须先确认**

```
用户：可以
助手：好的，开始执行修改...
```
