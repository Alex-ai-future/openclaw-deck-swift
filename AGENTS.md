## 🛑 核心规则：修改前必须确认

**在任何修改操作之前，必须遵循以下流程：**

```
1. 分析需求 → 2. 制定计划 → 3. 向用户确认 → 4. 获得批准 → 5. 执行修改
```

### 具体要求：

1. **不要直接修改文件**
   - 即使是很小的改动，也要先告知用户
   - 说明你要改什么、为什么改、怎么改

2. **提供完整的修改计划**
   - 列出所有要修改的文件
   - 说明每处修改的内容
   - 解释修改的原因和预期效果

3. **等待用户批准**
   - 用户说"可以"、"好的"、"批准"等明确表示后，才能执行
   - 如果用户有疑问，先解答再继续

4. **修改后汇报**
   - 完成后告知用户修改已完成
   - 如有需要，说明如何验证修改效果

5. **禁止直接修改代码** ⚠️
   - 任何代码修改前，必须先提供**逻辑严密的修改计划**
   - 计划必须包含：问题分析、修改方案、影响范围、预期效果
   - 没有计划的修改请求，先制定计划再执行
   - 这是保护项目的第一道防线

## 📋 工作流程

### 进入项目时：
- [ ] 读取本 `AGENTS.md` 文件
- [ ] 了解项目结构和当前状态
- [ ] 等待用户指令

### 收到任务时：
- [ ] 分析任务需求

- [ ] 执行 git 操作（commit/push 等）→ ⚠️ **必须先确认**

## 🔒 编译锁规则（多 AI 协作）

**问题：** 多个 AI 同时编译会导致 `build.db` 数据库锁定冲突

**解决方案：** 构建脚本已自动添加等待逻辑

### 编译前检查

脚本会自动检查 `build.db` 是否被锁定：
- 如果锁定：等待最多 60 秒（可配置）
- 等待期间：每 2 秒显示进度
- 超时后：退出并提示手动清理

### 自定义等待时间

```bash
# 设置更长等待时间（秒）
export MAX_BUILD_WAIT=120

# 运行构建
bash script/build_macos.sh
```

### 如果超时

```bash
# 手动清理构建缓存
rm -rf build/macos/DerivedData
rm -rf build/ipados/DerivedData

# 或者关闭所有 Xcode 进程
killall Xcode
```

### AI 协作建议

1. **编译前询问** - "现在可以编译吗？"
2. **避免同时编译** - 等另一个 AI 完成
3. **使用不同构建目录** - 如需频繁并发编译
