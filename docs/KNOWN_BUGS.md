# 已知 Bug 列表

本文档追踪 OpenClaw Deck Swift 项目中的已知问题。

## 文档目的

- **记录已知问题** - 已发现但尚未修复的 bug
- **追踪处理进度** - 通过状态字段了解每个 bug 的处理情况
- **会话协作** - 通过会话标识避免多个代理同时处理同一个 bug
- **提供复现步骤** - 帮助开发者快速定位问题

---

## Bug 状态说明

| 状态 | 说明 |
|------|------|
| 📋 待处理 | Bug 已记录，等待认领 |
| 🔧 处理中 | 已有代理认领并正在修复 |
| ✅ 已完成 | Bug 已修复，待验证 |
| ❌ 已关闭 | Bug 已验证修复或确认为非问题 |

## 字段说明

| 字段 | 说明 | 格式示例 |
|------|------|----------|
| 会话标识 | 区分哪个会话在处理（多会话协作时使用） | `session-1`、`webchat-A` |
| 认领时间 | 会话开始处理的时间 | `2026-02-28 10:30` |
| 最后更新 | 最后状态变更的时间 | `2026-02-28 10:35` |

---

## Bug 认领流程

**代理在开始修复 bug 前必须执行以下步骤：**

```
1. 读取 docs/KNOWN_BUGS.md → 2. 选择一个"待处理"状态的 bug → 3. 更新该 bug 的状态为"处理中" → 4. 设置"会话标识"为当前会话 → 5. 开始修复
```

### 具体规则：

1. **认领前检查**
   - 确认 bug 状态是"待处理"
   - 确认没有其他代理正在处理（状态不是"处理中"）

2. **认领操作**
   - 修改 bug 条目：
     - `状态`: 待处理 → 🔧 处理中
     - `会话标识`: [空] → [当前会话标识]（用于区分哪个会话在处理）
     - `认领时间`: 添加当前时间
     - `最后更新`: 添加当前时间

3. **修复完成后**
   - 更新 bug 状态：🔧 处理中 → ✅ 已完成
   - 添加 `修复说明` 字段
   - 添加 `修复时间`
   - 更新 `最后更新` 为当前时间

4. **验证关闭**
   - 用户验证修复后
   - 状态更新为：✅ 已完成 → ❌ 已关闭
   - 更新 `最后更新` 为当前时间

### 示例

```markdown
### BUG-001: 问题标题

- **状态**: 🔧 处理中
- **会话标识**: session-1
- **认领时间**: 2026-02-28 10:30
- **最后更新**: 2026-02-28 10:30
```

### BUG-001: SessionListView.swift - iOS 会话列表视图问题

- **ID**: BUG-001
- **标题**: SessionListView.swift - iOS 会话列表视图问题
- **涉及文件**: `SessionListView.swift`
- **状态**: ✅ 已完成
- **会话标识**: webchat-A
- **认领时间**: 2026-02-28 10:47
- **最后更新**: 2026-02-28 10:55
- **修复时间**: 2026-02-28 10:55
- **严重程度**: 高
- **平台**: iOS (iPhone)
- **报告日期**: 2026-02-28

**包含的子问题：**

#### 1.1 会话列表不显示
- **描述**: 在 iPhone 上运行时，会话列表视图无法渲染或显示任何会话记录。列表显示为空或完全不加载。
- **根本原因**: `SessionListView` 缺少 `.task` 自动连接逻辑，即使有保存的凭证也不会自动连接 Gateway。
- **修复方案**: 添加 `.task` 修饰符，在视图加载时自动连接 Gateway。

#### 1.2 设置按钮无响应
- **描述**: 在 iPhone 的会话列表设置视图中，点击取消或确定按钮时，按钮不响应触摸事件。点击后没有任何操作。
- **根本原因**: `SettingsView` 的 `onClose` 闭包未传递，导致按钮点击无反应。
- **修复方案**: 添加 `onClose` 闭包，并在所有操作完成后关闭设置页面。

#### 1.3 工具栏显示不完整
- **描述**: 会话列表视图只显示 2 个工具栏项目，但应该有更多可用工具。
- **根本原因**: `SessionListView` 缺少同步按钮和排序按钮（与 `DeckView` 不一致）。
- **修复方案**: 添加同步按钮和排序按钮，与 `DeckView` 保持功能一致。

**修复说明**:
1. 添加 `showingSortSheet` 和 `showingSyncAlert` 状态变量
2. 添加同步按钮（带旋转动画）和排序按钮
3. 添加 `handleSync()` 函数处理同步逻辑
4. 添加 `.task` 修饰符实现自动连接 Gateway
5. 修复 `SettingsView` 的 `onClose` 闭包缺失问题

**临时方案**: 暂无

### BUG-002: DeckViewModel.swift - iOS 会话列表手动添加会话后无法导航

- **ID**: BUG-002
- **标题**: DeckViewModel.swift - iOS 会话列表手动添加会话后无法导航
- **涉及文件**: `DeckViewModel.swift`
- **状态**: 📋 待处理
- **会话标识**: 
- **认领时间**: 
- **最后更新**: 2026-02-28 10:45
- **严重程度**: 高
- **平台**: iOS (iPhone)
- **报告日期**: 2026-02-28
- **视图**: 会话列表视图 (DeckView)

**问题描述**:
在 iOS 平台的会话列表视图中，手动添加新会话到列表后，会话显示在列表中但点击它无法导航到对应的会话详情页面。点击操作无效。

**复现步骤**:
1. 在 iPhone 设备或模拟器上构建并运行应用
2. 导航到会话列表视图
3. 手动添加一个新会话到会话列表
4. 观察到新会话显示在列表中
5. 点击新添加的会话
6. 观察到无法导航到会话详情页

**备注**: 这表明手动添加的会话可能缺少必要的导航数据、绑定，或者点击手势处理器没有正确配置动态添加的项目。这是 iOS 平台会话列表视图特有的问题。

**临时方案**: 暂无
