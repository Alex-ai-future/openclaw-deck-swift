# 使用示例

本文档通过真实的对话示例，展示如何使用 OpenClaw Deck 多会话聊天客户端提高工作效率。

---

## 核心场景：同时处理多个任务

OpenClaw Deck 的最大优势是**支持多个独立会话**，让你可以同时处理不同的任务，而不会互相干扰。

### 典型工作流

```
上午 9:00  →  会话 1 [代码开发]：开发新功能
上午 9:30  →  会话 2 [Bug 修复]：修复紧急问题
上午 10:00 →  切换回会话 1：继续开发
上午 10:30 →  会话 3 [文档写作]：编写使用说明
```

每个会话都有独立的上下文，AI 会记住每次对话的内容，随时切换都能继续。

---

## 示例 1：多任务并行处理

### 会话 1 - 代码审查

```
用户：帮我看看这个 Swift 视图的代码有什么问题

助手：分析了代码，发现 3 个潜在问题：

1. 第 45 行可能空指针 - 访问数组前没有检查是否为空
2. 第 78 行逻辑判断有误 - 条件判断应该用 || 而不是 &&
3. 第 120 行可以优化性能 - 这个计算可以提到循环外

需要我详细解释哪个问题？
```

### 会话 2 - 写文档

```
用户：帮我写一个使用示例文档，用对话形式

助手：好的，文档结构应该是：

1. 快速开始 - 如何连接和创建会话
2. 日常对话示例 - 基本使用方法
3. 多会话协作示例 - 核心功能展示
4. 使用技巧 - 会话组织建议

我现在开始写，写完给你看。
```

### 会话 3 - 技术调研

```
用户：查一下 SwiftUI 键盘处理的最新方案

助手：搜索了最新资料，iOS 15+ 推荐使用：

1. .safeAreaInset 处理输入框 - 自动避让键盘
2. ScrollView 配合 .ignoresSafeArea - 内容自动滚动
3. 不需要手动监听键盘通知 - 系统自动处理

需要我详细解释哪个方案？
```

**💡 效率提升：** 3 个任务同时进行，不需要等待一个完成再开始下一个。

## 示例 2：长期项目协作

### 第一天：项目启动

```
用户：我在开发一个 SwiftUI 应用，是多会话聊天客户端

助手：很好的项目！我帮你记录一下关键信息：

- 技术栈：Swift + SwiftUI
- 平台：macOS + iOS + iPadOS
- 核心功能：多会话聊天、Gateway 连接

接下来需要我帮你做什么？
```

### 第二天：继续开发

```
用户：昨天说的那个项目，今天继续

助手：好的，我记得你的 SwiftUI 聊天客户端项目。

昨天我们确定了技术栈和功能，今天要做什么？
- 实现某个具体功能？
- 修复已知问题？
- 还是优化现有代码？
```

### 第三天：问题修复

```
用户：滚动到底部的功能有问题，点击按钮没反应

助手：我来分析一下滚动逻辑...

问题找到了：
1. 滚动代理没有正确保存
2. 点击按钮时 ID 相同不会触发变化

我有个修复方案，要看看吗？
```

**💡 效率提升：** AI 记住项目上下文，不需要每次都重新解释。

## 示例 3：代码修改流程

### 步骤 1：提出需求

```
用户：过滤掉 Gateway 返回的空消息

助手：好的，我分析一下需要修改的地方：

1. GatewayClient.swift - 在获取历史消息时过滤
2. SessionColumnView.swift - 可以删除视图层的空消息判断

修改计划：
- 在数据层添加空消息检查
- 只过滤 assistant 角色的空消息
- user 消息即使是空也保留

可以执行吗？
```

### 步骤 2：用户批准

```
用户：可以

助手：好的，开始执行修改...

✅ 修改完成：
- GatewayClient.swift - 添加空消息过滤
- SessionColumnView.swift - 删除冗余判断

要提交代码吗？
```

### 步骤 3：提交确认

```
用户：提交

助手：好的，建议的提交信息：

[fix] 在数据层过滤 Gateway 返回的空消息

修改的文件：
- Services/GatewayClient.swift
- Views/SessionColumnView.swift

确认提交吗？
```

**💡 效率提升：** 修改前给计划，完成后等确认，安全可控。

## 使用技巧

### ✅ 推荐做法

| 场景 | 建议 |
|------|------|
| 不同项目 | 用不同会话 |
| 长期任务 | 用独立会话 |
| 敏感信息 | 不要告诉 AI |
| 代码修改 | 等 AI 给计划再批准 |

### ❌ 避免做法

| 场景 | 问题 |
|------|------|
| 所有事用一个会话 | 上下文混乱，AI 记不住 |
| 频繁切换主题 | AI 容易混淆 |
| 直接让 AI 提交代码 | 不安全，应该先检查 |

## 常见问题

**Q: 多个会话会互相干扰吗？**

A: 不会。每个会话有独立的上下文和记忆，AI 不会混淆不同会话的内容。

---

**Q: 如何删除不需要的会话？**

A: 长按会话或点击菜单，选择"删除会话"即可。

---

**Q: AI 会自己修改我的文件吗？**

A: 不会。AI 修改文件前会先给计划，等你批准后才执行。

---

**Q: 如何快速找到之前的对话？**

A: 每个会话会保留完整的对话历史，点击会话即可查看。

---

## 总结

OpenClaw Deck 的核心价值：

1. **并行处理** - 多个任务同时进行，不需要等待
2. **上下文独立** - 每个会话记住自己的主题
3. **快速切换** - 点击即可继续上次的对话
4. **安全可控** - 修改文件前先问，不会擅自行动

用好这些特性，让你的工作效率翻倍！
